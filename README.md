
# 階層フォルダのファイル名フラット化・復元ツール（BJB-PathFlattener）

> **注意:**
> 本ツールは「フォルダ階層をファイル名に埋め込んでフラット化」し、あとから元のディレクトリ構造に復元できるツールです。アップロードやクラウド連携機能はありません。
>
> **区切り文字・エスケープ仕様について**
> - 区切り文字（デフォルト: __ ダブルアンダースコア）やエスケープ文字列（___ など）と「完全に同じ文字列」がファイル名やディレクトリ名に含まれている場合、正しくフラット化・復元できません。
>   - 例: ファイル名そのものが「__」や「___」だけの場合や、区切り・エスケープ文字列が連続して現れる場合は不可逆になる可能性があります。
> - 通常の日本語・英数字・記号ファイル名では問題ありませんが、特殊な命名規則の場合はご注意ください。
>
> **OSの保護機能・アンチウイルスソフトについて**
> - Windows Defenderや他のアンチウイルスソフト、SmartScreenなどの保護機能により、
>   - 「レアなファイル」「不明な発行元」「PCを保護しました」等の警告が表示される場合があります。
>   - 場合によっては実行がブロックされることもあります。
> - これはウイルスではなく、配布直後や利用実績が少ない新規バイナリに対する一般的な挙動です。
> - **信頼できる入手元（公式リポジトリ等）からダウンロードした場合のみ、警告画面の「詳細情報」→「実行」や「許可」等を選択してご利用ください。**
> - 不安な場合は、ウイルス対策ソフトでスキャンしてからご利用ください。

## 概要
研究データや画像データなど、階層フォルダ内の大量ファイルを「1つのフォルダにまとめて管理・アップロードしたい」場合に便利なツールです。

本ツールは、
- フォルダ階層を「__（ダブルアンダースコア）」区切りでファイル名に埋め込むことで、全ファイルを1ディレクトリにフラット化
- 元の「__」や「___」などの記号もエスケープして可逆性を担保
- 変換後ファイル群やfilemap（CSV/JSON）から、元のディレクトリ構造を正確に復元
- GUI（Tkinterベース）で直感的に操作可能
- バイナリ配布（PyInstaller）にも対応


例えば、実際の研究データや画像データの場合：

| 元のディレクトリ/ファイル名         | フラット化後のファイル名                |
|:------------------------------------|:---------------------------------------|
| 観察データ/分析/画像1.tif           | 観察データ__分析__画像1.tif            |
| 観察データ/分析/画像__A.tif         | 観察データ__分析__画像___A.tif         |
| 観察データ/___特殊/データ.txt        | 観察データ___UNDERSCORE___特殊__データ.txt |
| サンプル/EDS/測定結果.dat           | サンプル__EDS__測定結果.dat            |
| サンプル/EDS/___/test.txt           | サンプル__EDS___UNDERSCORE____test.txt |

このように、パス情報をファイル名に埋め込んで1フォルダに集約できます。

**典型的な用途例**
- 階層付きデータを一括アップロードしたい（外部システムが階層非対応でもOK）
- 画像・分析データなどを「元の階層ごと」安全に保存・再構成したい

Python 3.9+ 対応。Windowsバイナリ（BJB-PF.exe）も同梱。

## 主な機能（現行コード準拠）
- 階層フォルダ内の全ファイルを「__（ダブルアンダースコア）」区切りでファイル名にパス情報を埋め込んでフラット化
  - 元ファイル名・ディレクトリ名に「__」や「___」が含まれる場合はエスケープ（___や___UNDERSCORE___）
  - 区切り・エスケープ仕様は `flatten_app/flattener/logic.py` の `flatten_filename`/`restore_flattened_filename` に準拠
- ZIP化推奨拡張子を含むディレクトリは自動でZIPファイル化（GUIで拡張子指定可）
- 除外拡張子・ZIP推奨拡張子はGUIで編集・保存可能
- filemap（CSV/JSON）を自動出力し、元の階層構造を100%復元可能
- filemapがなくても「ファイル名ベース」で階層復元（拡張子変換後もディレクトリ再現可）
- ZIP化/除外指定はツリービューで直感的にON/OFF切替
- プログレスバー・統計・色分け・進捗ラベルをリアルタイム表示
- 入出力フォルダ・設定は自動保存
- ヘルプボタン（❓）でアプリ内ガイドをいつでも参照可能
- バイナリ実行時はデバッグファイル出力なし（`sys._MEIPASS`判定）
- GUI（Tkinter）専用（CLIは未実装）

## 使い方（バイナリ・GUI専用）

### 起動方法
BJB-PF.exe を実行してください。

### フラット化手順
1. 「入力フォルダ」「出力フォルダ」を選択
2. [スキャン]ボタンで内容を確認
3. 必要に応じてZIP化/除外指定や拡張子設定を編集
4. 「フラット化実行」ボタンでフラット化を開始
5. 完了後、出力フォルダにフラット化ファイル・filemap.csv等が生成されます

### 復元手順
1. [復元]モードに切り替え
2. 「入力フォルダ」（フラット化済みファイル群）と「出力フォルダ」を選択
3. filemap優先/ファイル名推測・ZIP展開ON/OFFを選択
4. 「復元実行」ボタンで元の階層に復元

※ 詳細な使い方ガイドはアプリ内ヘルプ（❓ボタン）からも参照できます

## データ種別ごとのZIP化活用例・ZIP化推奨拡張子の役割

写真データとEDS分析データなど、異なる性質のデータが混在する場合の活用例です。

- 写真データ（例：JPG, PNGなど）は単体で意味があるため、フラット化しても個別ファイルとして扱えます。
- 一方、EDS分析データなどは「ディレクトリ単位でひとまとまり」として意味を持つため、フラット化時にそのディレクトリごとZIP化してまとめておくのが有効です。
- このツールでは、ZIP化推奨拡張子（例：EDSデータ特有の拡張子）が含まれるディレクトリを自動的にZIP化対象として整理できます。
- これにより、単体で意味を持つファイルと、ディレクトリ単位で管理すべきデータを適切に分離してフラット化・復元できます。

## ファイル名ベース復元の実用例（拡張子変換後の階層復元）

本ツールでフラット化したファイルを外部システム等にアップロードした際、
システム側でファイルの拡張子や内容が変換される場合があります（例: `aaaaaa.dm3` → `aaaaaa.png`）。
この場合でも、ファイル名にパス情報が埋め込まれているため、
ダウンロードした変換後ファイル群（例: `sample__exp1__aaaaaa.png`）を本ツールの「ファイル名ベース復元」機能で
元のディレクトリ構造に再配置できます。

### 典型的な流れ

1. 元ファイル: `sample/exp1/aaaaaa.dm3`
2. フラット名: `sample__exp1__aaaaaa.dm3`
3. 外部システムで `aaaaaa.dm3` → `aaaaaa.png` などに変換（ファイル名は維持）
4. 取得ファイル: `sample__exp1__aaaaaa.png`
5. 本ツールで復元 → `sample/exp1/aaaaaa.png` のように階層を再現

### ポイント
- 拡張子が変わっても、ファイル名ベースで階層復元が可能
- filemapがなくてもディレクトリ構造だけは正確に再現できる
- 「ファイル名維持＋拡張子変換」なシステムとの相性が非常に良い

### 注意
- 元の拡張子（例: `.dm3`）はfilemapがないと復元できませんが、ディレクトリ構造はファイル名から復元できます
- 変換後のファイル（`.png`, `.jpg`など）は、元の階層に自動で整理できます

## フラット名生成・復元仕様（v0.2.4～）

- パス区切りは「__」（ダブルアンダースコア）
- 元の「__」は「___」（トリプルアンダースコア）にエスケープ
- 元の「___」は「___UNDERSCORE___」に一時エスケープ
- 日本語や記号はエンコードせず可読性重視
- 復元時は逆変換（`restore_flattened_filename`）で元パスを再現

例：

| 元パス                | フラット名                       |
|:----------------------|:---------------------------------|
| foo/bar/baz.txt       | foo__bar__baz.txt                |
| foo__bar/baz.txt      | foo___bar__baz.txt               |
| foo___bar/baz.txt     | foo___UNDERSCORE___bar__baz.txt  |
| データ/サンプル_01.txt | データ__サンプル_01.txt           |
| dir/___/file.txt      | dir___UNDERSCORE____file.txt      |

※ エスケープ文字列・区切り文字は設定で変更可能
